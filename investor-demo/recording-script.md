# 猫小伴投资方 Demo 录屏脚本

目标时长：约 3 分钟
录制方式：打开 `investor-demo/index.html`，点击“开始 3 分钟演示”，按页面自动推进录制。需要手动强调某个片段时，可用方向键切换段落，或用 `Space` 暂停。

## 0:00-0:20 产品定位

画面：Hero 首屏，猫小伴在模拟桌面边缘轻微呼吸，展示“工作中：安静陪伴”的状态。

操作：点击“开始 3 分钟演示”，让页面进入录屏模式。

旁白：猫小伴是一个 macOS 桌面电子宠物。它不是单纯会动的小猫，而是一个懂工作状态、低打扰、本地隐私优先的桌面陪伴 App。投资方第一眼需要看到的是：它能长期停留在用户桌面边缘，成为日常工作流里的陪伴入口。

## 0:20-0:55 工作陪伴

画面：进入工作场景，模拟窗口轻微移动，小猫保持在桌面边缘。

操作：让自动演示停留在工作段落；如需强调，可短暂停顿在“小猫保持在边缘”的画面。

旁白：用户工作时，猫小伴会安静待在一边。它会结合前台应用、窗口覆盖率和空闲状态，尽量避开工作窗口，降低动作频率，不把陪伴变成打扰。这个场景解决的是桌宠产品最核心的矛盾：既要有存在感，又不能抢走注意力。

## 0:55-1:25 休闲动作

画面：展示探头、散步、伸懒腰、睡觉等真实动作 GIF。

操作：保持页面自动播放，让四个动作资产同时进入画面。

旁白：当用户休闲或空闲时，它会更活跃。散步、探头、伸懒腰、睡觉这些动作来自当前项目里的真实资产，说明产品不是概念页，而是已经有可演示的视觉行为。后续也可以继续扩展成宠物、动作包和皮肤体系。

## 1:25-1:55 轻提醒

画面：提醒气泡出现，小猫探头，气泡里展示“喝口水吧”和完成、稍后、跳过三个动作。

操作：自动演示会触发“稍后”状态；如需录制互动效果，可手动点击“完成”，让小猫切换到庆祝动作。

旁白：猫小伴也承担轻提醒角色。喝水、休息、久坐、睡觉提醒都可以配置。重点是低打扰：安静工作、会议或全屏时，提醒会延后，不会强行打断用户。它把健康提醒从闹钟式打断，变成更柔和的桌面互动。

## 1:55-2:25 状态感知

画面：流程图从前台应用、窗口覆盖率、行为判断推进到小猫动作，调试面板显示状态、动作、提醒队列和原因。

操作：让流程高亮自动轮转，录到“前台应用 -> 窗口覆盖率 -> 行为判断 -> 小猫动作”的完整链路。

旁白：它的底层不是随机动画，而是一套轻量的桌面行为感知系统。应用分类、窗口覆盖、空闲时间和时间节律，都会影响它下一步做什么。投资价值在这里：猫小伴不是一次性好玩的挂件，而是可以持续理解用户桌面状态的入口。

## 2:25-2:35 本地隐私

画面：隐私边界三栏，展示“读取”“本地保存”“不上传”。

操作：保持镜头稳定，让三栏文字清楚可读。

旁白：当前版本本地读取应用名称、窗口几何和空闲时间，本地保存设置与提醒。不上传屏幕内容，不读取窗口正文，也不联网同步数据。这个边界让它更适合长期驻留在用户桌面。

## 2:35-3:00 投资亮点

画面：投资亮点卡片，展示情绪陪伴、桌面入口、资产扩展、商业路径。

操作：停留在最后一屏，录到庆祝动作和资产扩展卡片。

旁白：猫小伴验证的是一个长期停留在桌面边缘的陪伴入口。它可以从桌面宠物扩展到情绪陪伴、办公健康提醒、个性化皮肤、AI 对话和品牌联名。当前版本已经能跑，下一步是把体验打磨成可规模化的消费级产品。
