# 猫小伴跨对话笔记约定

这个目录用于在同一个项目的不同 Codex 对话之间共享信息。

## 使用规则

- 任何猫小伴相关的新想法、内测反馈、功能灵感，如果需要跨对话保留，先写入本目录的 Markdown 文件。
- 不把长期规划只留在聊天窗口里。
- 重要反馈同步到项目根目录的 `INTERNAL_QA_NOTES.md`。
- 单个主题优先使用独立文件，例如 `功能迭代灵感记录.md`。

## 推荐流程

在另一个 Codex 对话窗口里直接发送：

```text
请把本对话里关于猫小伴的功能迭代灵感，整理同步到：
/Users/zhanghz/Documents/猫小伴-电子宠物/docs/notes/功能迭代灵感记录.md

要求：
- 按功能方向分组。
- 保留原始想法，但改写成可执行条目。
- 标注优先级：P0 / P1 / P2 / P3。
- 标注阶段建议：V1.0.1 / V1.1 / V1.2 / V1.3 / Backlog。
- 最后补一个“可同步到 INTERNAL_QA_NOTES.md 的摘要”。
```

同步完成后，当前项目内其他对话就可以直接读取这些文件，不需要访问 Codex 其他窗口。
