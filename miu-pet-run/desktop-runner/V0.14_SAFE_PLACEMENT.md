# V0.14 Safe Placement

Goal: place Miu more precisely instead of always snapping to the lower-right corner.

Borrowed idea from DockCat:

- Treat screen geometry as a first-class input.
- React to display / Space changes.
- Use visible screen area instead of raw full screen area.

Implemented:

- Uses `NSScreen.visibleFrame` so Miu avoids:
  - Dock
  - menu bar
  - reserved screen edges
- Reads frontmost window geometry with `CGWindowListCopyWindowInfo`.
- Converts CoreGraphics top-left window bounds into AppKit-style coordinates for placement.
- Generates multiple candidate positions:
  - lower-right
  - lower-left
  - upper-right
  - upper-left
  - left / right middle
  - top / bottom middle
- Scores candidates by:
  - overlap with the active window plus margin
  - distance from the active window
  - stable preferred corner order
- Picks the safest candidate when auto-positioning.
- Repositions after:
  - active app changes
  - Space changes
  - display / Dock layout changes

Notes:

- User-pinned position still wins.
- Leisure walking still stays on the visible lower edge.
- Fullscreen / presentation-sized windows still hide Miu as before.
