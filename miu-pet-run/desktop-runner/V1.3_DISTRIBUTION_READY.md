# V1.3 Distribution Ready

Goal: make local release packaging ready for public distribution handoff.

Completed locally:

- Release scripts already support app, zip, DMG, signing, notarization, and verification flows.
- Documentation now calls out Gatekeeper, Developer ID signing, notarization, and GitHub Release handoff.
- Local ad-hoc builds remain supported for development.
- `build-release.sh` builds the app, syncs `/Applications/猫小伴.app` with `OPEN_AFTER_INSTALL=0`, then builds the DMG.
- GitHub Desktop is the preferred upload path on this machine when terminal Git cannot reach GitHub.

External blockers:

- Apple Developer ID Application certificate must be installed in the local keychain.
- Notary credentials must be configured with `xcrun notarytool store-credentials`.
- A GitHub Release requires repository access and a chosen release tag.
- Terminal `git push origin --tags` may need a stable network because GitHub Desktop does not always publish local tags with the branch.

Recommended final release flow:

```bash
cp miu-pet-run/desktop-runner/release.env.example miu-pet-run/desktop-runner/release.env
$EDITOR miu-pet-run/desktop-runner/release.env
miu-pet-run/desktop-runner/build-release.sh
miu-pet-run/desktop-runner/verify-release.sh miu-pet-run/dist/猫小伴.app
miu-pet-run/desktop-runner/verify-dmg.sh miu-pet-run/dist/猫小伴.dmg
```

GitHub upload note:

- Use GitHub Desktop for branch upload when terminal Git reports `SSL connection timeout` or cannot connect to `github.com:443`.
- After the branch is synced, push tags separately when terminal network is healthy:

```bash
git push origin --tags
```
