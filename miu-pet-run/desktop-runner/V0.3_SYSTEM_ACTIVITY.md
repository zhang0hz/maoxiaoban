# V0.3 System Activity Detection

Goal: Miu judges work/leisure from whole Mac activity, not Codex-only events.

## Signals

- Frontmost app: `NSWorkspace.shared.frontmostApplication`
- Active app changes: `NSWorkspace.didActivateApplicationNotification`
- Front-window coverage: `CGWindowListCopyWindowInfo`
- User idle time: `CGEventSource.secondsSinceLastEventType`
- Beijing-time phase: existing day/night rhythm

## Classifier

Auto mode uses this priority:

1. Manual menu mode overrides all.
2. Long idle + night/sleepy -> sleep.
3. Work apps -> work.
4. Communication/meeting apps -> quiet work.
5. Leisure apps -> leisure, or sleep at night.
6. Browser -> work during work hours or large focused window; leisure otherwise.
7. Very large front window -> quiet work.
8. Fallback -> Beijing-time rhythm.

## Privacy

- Local only.
- No screen upload.
- No network call.
- No Accessibility permission yet.
- No window title or browser tab title reading yet.

## Current UI

Menu shows:

- `状态：<模式> · <应用>`
- `原因：<classifier reason>`
- `当前应用分类` override menu

Menu bar tooltip also shows current classifier reason and bundle id.

## V0.3.1 Overrides

User can mark current frontmost app:

- work
- leisure
- communication
- neutral

Storage:

```text
~/Library/Application Support/Miu/app-classifications.json
```

Override priority:

1. Current app override.
2. Built-in bundle id category.
3. Built-in app-name keyword category.
4. Browser/time/window fallback.
5. Beijing-time fallback.
