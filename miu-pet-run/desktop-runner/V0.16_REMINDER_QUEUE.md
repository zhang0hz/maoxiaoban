# V0.16 Reminder Queue

Goal: make reminders polite when Miu is in quiet/fullscreen states.

Implemented:

- Deferred reminder queue:
  - fullscreen or quiet work states no longer fire a bubble immediately
  - due reminders are recorded in `deferredIds`
  - when the screen is no longer blocked, queued reminders replay first
- Reminder history:
  - records triggered, deferred, replayed, completed, snoozed, skipped, and dismissed events
  - keeps the newest 120 events
- Menu and overview summaries:
  - queue status
  - today completion/skipped count
  - today history count

Persistence:

- Reminder queue and history are saved in:

```text
~/Library/Application Support/Miu/reminders.json
```

Notes:

- Existing reminder JSON remains compatible.
- Active reminders are suppressed while fullscreen/quiet is detected, so Miu does not cover work.
