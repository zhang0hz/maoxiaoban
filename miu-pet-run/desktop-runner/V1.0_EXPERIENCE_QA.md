# V1.0 Experience QA

Goal: verify 猫小伴 feels safe, calm, and showable before more feature work.

## Automated Smoke

Run:

```bash
miu-pet-run/desktop-runner/smoke-test.sh
```

Checks:

- App bundle builds.
- Behavior-layer tests pass.
- All action frames pass color audit.
- Shell scripts parse.
- Python helper scripts compile.
- App `Info.plist`, executable, frames, app icon, and status icon exist.

## Manual Experience Pass

Work scene:

- Open code editor / browser / document windows.
- Confirm 猫小伴 stays near safe edge and does not cover the active work area.
- Confirm working state selects calm actions: `loaf`, `sleep`, `groom`, `blink`, or `purr`.
- Open `设置...` -> `调试`; confirm app classification and reason match current app.
- Click `复制诊断信息`; paste content into any text field and confirm it contains recent behavior log.

Leisure scene:

- Open a media or casual browser window.
- Leave mouse idle for several minutes.
- Confirm leisure actions can use peek / walk / tail-sway without covering the main window.
- Confirm walking direction matches movement direction.

Reminder scene:

- Open `设置...` -> `提醒`.
- Temporarily lower a reminder interval.
- Confirm reminder bubble appears outside fullscreen / quiet state.
- Confirm `完成`, `稍后10`, `跳过`, and `关闭` work.
- Confirm deferred queue appears in Overview / Debug while fullscreen or quiet work blocks reminders.

Placement scene:

- Drag 猫小伴 manually.
- Confirm manual position pins.
- Use `重置位置`; confirm auto-position returns.
- Test `右下`, `左下`, `右上`, `左上`.
- Test with Dock visible and hidden if available.

Night scene:

- In `行为`, switch to `睡觉`.
- Confirm sleep action and low movement.
- Return to `自动`.

Settings scene:

- Toggle `暂停动画`.
- Change `小 / 中 / 大`.
- Toggle `登录时自动启动猫小伴`.
- Quit and reopen; confirm settings persist.

## Pass Criteria

- No blocking of active work window during normal work.
- No color drift in triggered actions.
- No moonwalk during leisure walking.
- Reminder can be acted on without trapping focus.
- Debug diagnostics can explain any odd behavior.
- Build, smoke, and release verification pass.
