# V0.17 Behavior State Machine

Goal: make behavior decisions easier to extend without changing every branch in `updateBehavior()`.

Implemented:

- Added `BehaviorState`:
  - fullscreen
  - temporary reaction
  - night
  - morning
  - sleepy
  - work
  - leisure
  - idle
- Added `BehaviorDecision`:
  - selected action
  - minimum hold time
  - hide/show behavior
  - placement behavior
  - walking-placement guard
- Refactored behavior flow:
  - collect current context
  - check reminder queue
  - compute behavior decision
  - apply decision consistently

Kept Stable:

- Existing visual actions and timing remain the same.
- Leisure walking still keeps its current edge-walk placement.
- Fullscreen still hides Miu.
- Manual mode still overrides automatic mode.

Next:

- Split the state machine into a standalone file when the runner moves toward a larger app architecture.
- Add debug UI for current `BehaviorState` if needed.
