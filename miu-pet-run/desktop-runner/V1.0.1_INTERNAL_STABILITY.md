# V1.0.1 Internal Stability

Goal: tighten the internal build around daily-use friction before adding larger product features.

Implemented changes:

- Transparent sprite pixels no longer receive mouse clicks, so the invisible edge of the pet window is less likely to block work behind it.
- Status bar menu informational rows are truncated with full text kept in tooltips, preventing long reasons or reminder text from stretching the menu.
- Placement scoring now considers multiple visible front windows on the current screen instead of only the first eligible window.

Manual QA:

- Click near the transparent corners around the pet and confirm clicks reach the window behind when outside visible fur/body pixels.
- Right-click directly on visible pet pixels and confirm the control menu still opens.
- Create a long reminder message and confirm the menu remains narrow while the full text is available as a tooltip.
- Open multiple overlapping work windows and confirm auto-placement chooses a corner or edge that avoids the group better than before.
