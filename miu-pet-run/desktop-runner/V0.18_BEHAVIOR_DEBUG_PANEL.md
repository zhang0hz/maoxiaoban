# V0.18 Behavior Debug Panel

Goal: make behavior tuning observable before changing deeper heuristics.

Implemented:

- Added `设置...` -> `调试`.
- Added a manual refresh button that runs one behavior tick and updates the panel.
- Shows current behavior state and last behavior decision.
- Shows current animation action, last stable behavior action, and temporary action.
- Shows frontmost app name, bundle ID, local classification, resolved mode, manual override, quiet flag, and reason.
- Shows visible screen rect, front window rect, front-window coverage, and idle time.
- Shows placement status: auto-position, user-pinned state, placement preference, walking edge direction, and size scale.
- Shows reminder queue, deferred reminders, today's reminder history, Beijing time, and current day/night mode.
- Shows the latest 20 behavior changes with time, state, action, app, window coverage, idle time, and reason.
- Added `复制诊断信息`, which copies current state plus recent behavior log to the clipboard.

Code touched:

- `BehaviorModels.swift`: Chinese labels for `PetMode` and `PlacementPreference`.
- `BehaviorModels.swift`: `BehaviorLogEntry` for recent state changes.
- `MiuDesktopRunner.swift`: debug state fields and behavior log storage.
- `MiuRunner+Behavior.swift`: behavior loop now records debug snapshots and recent state changes.
- `MiuRunner+Settings.swift`: debug tab UI and sync helpers.

Usage:

1. Open the menu bar item `猫小伴`.
2. Choose `设置...`.
3. Open the `调试` tab.
4. Click `刷新调试信息` when you want an immediate fresh sample.
5. Click `复制诊断信息` when you want to paste a bug report or tuning sample.
