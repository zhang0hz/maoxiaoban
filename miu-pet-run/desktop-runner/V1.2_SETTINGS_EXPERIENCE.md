# V1.2 Settings Experience

Goal: make settings feel closer to a product surface while preserving diagnostics for development.

Implemented changes:

- Settings window is resizable with a minimum size.
- Overview now shows version information from `Info.plist`, including app version, build number, Bundle ID, and internal distribution status.
- Application classification moved under an `高级` tab label.
- Behavior settings now include `恢复推荐设置`, which restores mode, placement, animation, and size without deleting reminders or app classifications.
- Reminder edits now validate time and interval input before saving.
- Invalid reminder input reports a clear inline message and does not silently clamp or fallback.
- Existing debug panel remains available for development and QA.

Manual QA:

- Resize the settings window and confirm tabs remain usable.
- Enter `25:99` in a reminder time field and confirm saving is blocked with a message.
- Enter an interval below 5 or above 720 minutes and confirm saving is blocked.
- Confirm normal reminder edits still save and resync the editor fields.
- Confirm the overview version matches `CFBundleShortVersionString` and `CFBundleVersion`.
- Click `恢复推荐设置` and confirm reminders and app classification rules are not deleted.
