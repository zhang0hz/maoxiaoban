# V1.0 Release Prep

Product identity:

- Product name: 猫小伴
- Bundle ID: `com.zhanghz.maoxiaoban`
- Version: `1.0.0`
- Build: `100`
- App bundle: `猫小伴.app`
- Distribution zip: `猫小伴.zip`
- Data directory: `~/Library/Application Support/MaoXiaoBan`

Implemented in V1.0.1:

- Updated `Info.plist` with release name, bundle ID, version, category, and copyright.
- Renamed app bundle output from `Miu.app` to `猫小伴.app`.
- Renamed distribution output from `Miu.zip` to `猫小伴.zip`.
- Updated install script to install `/Applications/猫小伴.app`.
- Install script removes old `/Applications/Miu.app` to avoid duplicate apps.
- Runtime user-facing strings now use `猫小伴`.
- Local data moves to `Application Support/MaoXiaoBan`.
- First launch after upgrade migrates legacy `Application Support/Miu` into the new data folder if needed.

Implemented in V1.0.2:

- Added `release.env.example` for release-time signing and notarization settings.
- Added `entitlements.plist` for Hardened Runtime signing.
- Added `sign-release.sh`.
- Added `notarize-release.sh`.
- Added `verify-release.sh`.
- Added `build-release.sh`.
- `build-dist.sh` now signs the app bundle before zipping.
- Local builds use ad-hoc signing by default.
- Developer ID builds can be enabled by copying `release.env.example` to `release.env` and filling `SIGN_IDENTITY`.
- Notarization can be enabled with `NOTARIZE=1` after a `notarytool` keychain profile is created.

Implemented in V1.0.3:

- Added `build-dmg.sh`.
- Added `verify-dmg.sh`.
- `build-release.sh` now creates `猫小伴.app`, `猫小伴.zip`, and `猫小伴.dmg`.
- DMG includes:
  - `猫小伴.app`
  - `Applications` symlink
- `notarize-release.sh` now accepts a generic artifact and staples DMG artifacts.
- Added install, uninstall, and troubleshooting docs.

Implemented in V1.0.4:

- Added productized app icon based on the canonical 猫小伴 cat asset.
- Added `make-app-icon.py` and `build-icon.sh`.
- `build-app.sh` now bundles `AppIcon.icns`.
- `Info.plist` now declares `CFBundleIconFile`.
- Added `V1.0.4_ICON_POLISH.md`.

Implemented in V1.0.5:

- Split Swift code into focused files.
- Added `V1.0.5_CODE_SPLIT.md`.
- Updated `build-app.sh` to compile all top-level Swift files.

Release blockers still open:

- Developer ID signing certificate.
- Apple notarization.
- Gatekeeper verification on a clean Mac.

Next suggested step:
