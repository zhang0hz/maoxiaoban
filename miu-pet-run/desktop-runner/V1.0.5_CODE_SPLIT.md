# V1.0.5 Code Split

Goal: reduce release-maintenance risk without changing behavior.

Implemented:

- Split the former single `MiuDesktopRunner.swift` into focused Swift files:
  - `AppSupport.swift`: app identity, runner config, settings storage
  - `BehaviorModels.swift`: pet modes, placement preference, behavior decision types
  - `ReminderScheduler.swift`: reminder config, queue, history, scheduler
  - `SystemActivityClassifier.swift`: app classification and activity snapshots
  - `PetView.swift`: draggable transparent pet view
  - `MiuDesktopRunner.swift`: app delegate, windows, menus, behavior application
  - `main.swift`: executable entry point and asset resolution
- Updated `build-app.sh` to compile all top-level Swift source files.
- Further split runner extensions:
  - `MiuRunner+Placement.swift`: placement, front-window geometry, walking edge movement
  - `MiuRunner+Settings.swift`: settings window, reminder editor, settings sync
  - `MiuRunner+Menu.swift`: status bar item and control menu
  - `MiuRunner+ReminderBubble.swift`: reminder bubble panel and placement
  - `MiuRunner+Behavior.swift`: animation tick, behavior tick, state-machine decision, day/night action selection

Verification:

- App bundle build passes.
- Distribution app and zip build pass.
- Local ad-hoc code signing verification passes.
- Behavior-layer tests pass.
- Color audit passes.
- Shell scripts pass `bash -n`.
- Python scripts pass `py_compile`.

Notes:

- No business logic was intentionally changed.
- `MiuDesktopRunner.swift` now owns app lifecycle, command handlers, settings persistence, onboarding, and top-level runner state.
- Remaining useful cleanup: command handlers can move into a dedicated `MiuRunner+Commands.swift`.
